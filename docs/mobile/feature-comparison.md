## 2. 已知限制

### 2.1 Android 端限制

#### 系统限制

| 限制项 | 说明 | 解决方案 |
|--------|------|----------|
| 后台限制 | Android 8+ 对后台服务和广播的限制 | 使用 WorkManager + 前台服务 |
| 电池优化 | 系统可能杀死后台应用 | 引导用户关闭电池优化 |
| 通知渠道 | Android 8+ 必须配置通知渠道 | 已适配 |
| 存储权限 | Android 10+ 分区存储限制 | 使用 MediaStore API |
| 位置权限 | Android 10+ 后台位置需要特殊申请 | 按需申请 |
| 相机权限 | 实时权限申请 | 已适配 |

#### 功能限制

| 功能 | 限制说明 | 计划解决时间 |
|------|----------|--------------|
| 视频通话 | 目前仅支持 P2P，不支持中转 | v1.2.0 |
| 文件传输 | 单文件限制 100MB | v1.1.5 |
| 消息搜索 | 仅支持标题搜索，不支持内容 | v1.2.0 |
| 屏幕共享 | Android 不支持作为共享源 | 评估中 |
| 群组通话 | 暂不支持 | v1.3.0 |

### 2.2 桌面端限制

| 功能 | 限制说明 | 计划解决时间 |
|------|----------|--------------|
| 生物识别 | 依赖硬件支持 | v1.2.0 (Windows Hello) |
| 推送通知 | 需要常驻后台 | 使用 WebSocket 保活 |
| 移动端功能 | 部分功能不适用 | - |

## 5. 迁移指南

### 5.1 从桌面端迁移到 Android

#### 数据迁移

```
迁移步骤:
1. 在桌面端导出聊天记录
   设置 → 聊天 → 备份聊天记录 → 导出到云端

2. 在 Android 端登录同一账号

3. 恢复聊天记录
   设置 → 聊天 → 恢复聊天记录

注意:
- 媒体文件需要重新下载
- 部分桌面端设置不会同步
- 快捷键设置需要重新配置
```

#### 功能差异适应

| 桌面端习惯 | Android 替代方案 |
|------------|------------------|
| Ctrl + Enter 发送 | 直接点击发送按钮 |
| Ctrl + V 粘贴图片 | 长按输入框 → 粘贴 |
| 拖拽文件发送 | 点击附件按钮选择文件 |
| 多选消息 (Ctrl + 点击) | 长按消息 → 多选 |
| 全局搜索 (Ctrl + K) | 点击搜索图标 |

### 5.2 从 Android 迁移到桌面端

```
迁移步骤:
1. 在 Android 端启用云端备份
   设置 → 聊天 → 聊天记录备份 → 备份到云端

2. 在桌面端登录同一账号

3. 自动同步聊天记录

注意:
- 桌面端不支持生物识别，需设置密码
- 部分移动端功能在桌面端不可用
```
